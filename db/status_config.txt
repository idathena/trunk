// Status Change Configuration Database
//
// Defines configurations of status changes (SC).
//
// Structure of Database:
// SCType,Flag
//
// Legend for 'Flag' field (bitmask),
// 1    - SC cannot be removed by death.
// 2    - SC can be removed on logout.
// 4    - SC can be removed by Dispell.
// 8    - SC can be removed by Clearance.
// 16   - SC considered as debuff.
// 32   - SC cannot be reset by 'sc_end SC_ALL' and status change clear.
//
// Example:
// SC_ENDURE,3	// Endure status cannot be removed by death and cannot be saved (1 + 2)
